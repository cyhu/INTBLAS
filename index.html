IntBox 1.0 – A General Purpose Interval Basic Linear Algebra Toolbox

This package, IntBox, contains a C++ implementation that closely follows the 2002 Interval Basic Linear Algebra Subroutines (BLAS) standard for general, symmetric, triangular, banded, symmetric banded, and triangular banded matrices. It is ready to install on machines with GCC or Microsoft Visual Studio .NET 2003 or later.

Successful installation should result in the appropriate header files being added to the compiler's default include directory.

About IntBox

IntBox is an object-oriented interval basic linear algebra subroutines (BLAS) package in C++. The major reference for the design and implementation of this package is the proposed interval BLAS standard. The library consists of arithmetic, algebraic, utility functions, and set operations among intervals, interval vectors and interval matrices. These operations are implemented as methods of the seven template classes in this library. The template class IntervalVector is used for all vector operations. As per the standard there are six different matrix classes: IntervalMatrix, TriIntervalMatrix, SymIntervalMatrix, BandedIntervalMatrix, TriBandedIntervalMatrix and SymBandedIntervalMatrix.

The package is designed to be portable, well-tested, and robust with built-in error handling features. It can be easily embedded into an ANSI/ISO C++ environment to enable interval software development. Instructions on package installation, testing, usage, and sample applications are included below.

Boost

IntBox makes use of Boost's interval_lib library to provide the basic interval operations. Their are many good quality interval libraries out there, but we chose boost for a several of reasons. First, the Boost License is very flexible. This allows us to distribute the interval_lib library without having to distribute all of the much larger Boost package. Second, Boost is cross-platform compatible, which is one of the prime goals of the IntBox library. Lastly, Boost's interval library seems to be the most compatible with the proposed standard C++ interval class. Our plan is to eventually use the standard interval class when it made widely available. This means that when the change does occur it can be done with a minimum of effort.

You can learn more about Boost at www.Boost.org. Or to learn more about boost's interval library visit Interval Arithmetic Library section of Boost's website.

Installation

The library is easy to install. The installation procedures outlined here have been tested using GCC on Ubuntu Linux, and on Windows using Microsoft Visual Studio .Net.

Visual Studio

For quite a while we were issuing a binary installer to use with Visual Studio. But do to Microsoft putting out a new edition every couple of years and our researchers differing versions, we decided it was to difficult to take into account all the various version. So the installation is now done by hand.

  1. Download the zip file.
  2. Locate the installation directory for you edition of Visual Studio. This directory can usually be found by first going to the Program Files directory. The directory's name changes with each different version of Visual Studio, it is usually begins with "Microsoft Visual Studio" and then is suffixed with a version number or a year number.
  3. You will need to copy the contents of the src directory to the vc/include directory which is within the Visual Studio install directory.
  4. You can also copy the doc directory to a convenient place.
  5. To uninstall the library, just delete the files you copied.

Linux/Unix using GCC

To install this library you will need GCC and make. By default the header files are installed in /usr/include, and the documentation is installed in /usr/share/doc/IntBox-1.0. If these are not the paths you desire then you can make the appropriate changes to the configuration section of the Makefile. Note, you will need root permissions to install in the default location.

  1. Open a console
  2. Go to the package's gcc folder
  3. Type make install This will copy the header files to the compiler's include directory. By installing this library, the Boost interval_lib library is also installed. If you already have boost installed, that installation will not be overwritten.
  4. If you wish to create the testing application, type make test to create it.
  5. To uninstall the package type make uninstall Note, the uninstaller will not remove the Boost interval_lib library.

Other Platforms

The above installation procedures can be used on the platforms we use for testing. The library is tested using Ubuntu Linux 6.1, and Windows XP. If you wish to run the library on another system or using another compiler, the easiest way is probably to edit the Makefile for GCC to have it use your compiler and installation paths. Alternatively, you can simply copy the files and folders in the src folder to your project's or compiler's include directory. The code itself is written in ANSI/ISO standard C++, therefore as long as your compiler is fairly standards compliant, the code should function without error. Next, you will need to know if the library works. This is the purpose for including the unit tests with the distribution. If you are not using the Makefile, then simply build and link all the files found in the test directory into one executable.

Testing The Library

You can run the standard test cases for the library using the packaged tester application. For Visual Studio, simply open the solution file in the msvc folder. Then compile and run. On Unix like systems. Simply execute make test in the gcc folder.

The test application is used to run the standard test cases. The application is console based. From the appropriate directory type test -h for a listing of the available options.

The Tester

The application itself is a simple command line program. The tester simply outputs whether a test passes or fails. As mentioned there are 149 individual test cases. To make it simpler to observe the test result the output is split into two parts. First, is a category based output, this simply reports what category of tests is being worked and how many errors resulted. The second output reports on the pass/fail status of each test case. This output is very verbose, hence it is recommended that it be placed in a separate file. The -v and -m switches allow one to test only the vector or matrix classes. If these are the only switches used then a short test is performed. However, either of the next two switches causes the tester to go into long test mode. In long test mode the tester repetitively performs different test cases using different matrices. The -r switch specifies the number of repetitions, and the -d switch specifies the maximum size of the vectors and matrices to use.

Example:

> test -v -m dump.txt
This will cause the IntervalVector and IntervalMatrix classes to be tested and the output will be written to the file dump.txt, further the long tests will be skipped.

> test -r 40 -m
This will perform a long test where many of the cases will be tested forty times using different matrices, whose dimensions range from five twenty. Furthermore, all of the output will be printed on the screen.

Portability Testing

Portability tests are carried out by executing the tester on targeted platforms. The library has been tested on two processors, and three operating systems. It was mainly developed using Windows on an Intel Core Duo processor, using Microsoft Visual Studio .NET and GCC 3.4.4 running under CygWin. The library was also tested on Ubuntu Linux 6.1 on an Intel Pentium 4M using GCC 4.1.2.

Usage

The IntBox library is easy to use, it is simply a set of header files. Sample C++ programs in the sample directory are ready to be compiled and run. Below are the steps needed in order to make use of the IntBox library.

  1. Include the file <IntBox.hpp> in your source file(s).
  2. All classes, functions, global variables, etc. are defined in the intbox name space. So make the appropriate using declarations.
